Class BulkEditDataSource - extends DataSource.Local

BulkEditDataSource manages a YUI DataSource + diffs (insertions, removals, changes to values).

The YUI DataSource must be immutable, e.g., if it is an XHR datasource, the data must not change.

By using a DataSource, we can support both client-side pagination (all data pre-loaded, best-effort save allowed) and server-side pagination (load data when needed, only all-or-nothing save allowed). Server-side pagination is useful when editing a large amount of existing records or after uploading a large number of new records. (Store the new records in a scratch space, so everything does not have to be sent back to the client after parsing.) In the case of bulk upload, server-side validation will catch errors in unviewed records.

The responseSchema passed to the YUI DataSource must include a comparator for each field that should not be treated like a string. This comparator can either be 'integer', 'decimal', or a function which takes two arguments.

Configuration Attributes

ds - writeonce {DataSource}

REQUIRED. The original data. This must be immutable, i.e., the values must not change.

extractTotalRecords - writeonce {Function}

REQUIRED. The function to call to extract the total number of records from the response.

generateRequest - writeonce {Function}

REQUIRED. The function to convert the initial request into a request usable by the underlying DataSource. This function takes one argument: state (startIndex,resultCount,...).

generateUniqueId - writeonce {Function}

The function to call to generate a unique id for a new record. The default generates "bulk-edit-new-id-#".

startIndexExpr - writeonce {String}

OGNL expression telling how to extract the startIndex from the received data, e.g., .meta.startIndex. If it is not provided, startIndex is always assumed to be zero.

totalRecordsReturnExpr - writeonce {String}

OGNL expression telling where in the response to store the total number of records, e.g., .meta.totalRecords. This is only appropriate for DataSources that always return the entire data set.

uniqueIdKey - writeonce {String}

REQUIRED. The name of the key in each record that stores an identifier which is unique across the entire data set.
